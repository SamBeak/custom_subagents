# Example 1: 레이저 마킹기 출력 불안정 이슈 R&D 작성

## 시나리오

연구개발팀 리더가 주간 회의 전에 '장비 개발 이슈 문제점' 시트의 미작성 행을 일괄 작성하는 예시입니다. 시트에는 레이저 마킹기 관련 이슈 3건이 입력되어 있고, 이슈내용(K열)만 작성되어 있으며 R&D 영역(원인 분석~비고)은 비어 있는 상태입니다.

## 대화 흐름

### 사용자 입력

```
'장비 개발 이슈 문제점' 시트의 R&D 칸 비어있는 행 채워줘.
시트 ID는 1abcDEF...xyz 야.
```

### 에이전트 동작

1. **시트 메타 확인**
   - `mcp__google-sheets__list_sheets`로 '장비 개발 이슈 문제점' 시트 존재 확인
   - 1행 헤더 매핑: D열=장비 구분, K열=이슈내용, L=원인 분석, M=개선 대책, N=담당자, O=조치 예정일, P=조치 완료일, Q=검증 결과, R=비고

2. **대상 행 식별**
   - K열 값이 있고 L~R열 중 하나 이상 빈 행 → 3개 발견 (행 5, 7, 9)

3. **사용자 확인**

```
헤더 매핑 결과:
- 입력 컬럼: D(장비 구분), K(이슈내용)
- 출력 컬럼: L(원인 분석), M(개선 대책), N(담당자), O(조치 예정일),
            P(조치 완료일), Q(검증 결과), R(비고)
- 대상 행: 3개 (5, 7, 9행)

이대로 진행할까요?
```

사용자: "응"

4. **근거 자료 수집** (행 5 예시 - 레이저 마킹기 출력 불안정)
   - `Glob`으로 프로젝트에서 `**/laser*`, `**/marking*` 검색 → `firmware/laser_ctrl.c`, `docs/laser_spec.md` 발견
   - `Grep`으로 "PWM" 검색 → `firmware/laser_ctrl.c:142` 라인 식별
   - Obsidian `obsidian_simple_search` (query: "레이저 마킹기") → 일일업무보고 2026-04-15, 04-18 매칭
   - 보고서 본문에서 "PWM 듀티 30%에서 출력 변동 발견, 김철수 조사 중" 추출

5. **R&D 항목 작성** (행 5)
   - 원인 분석: `직접: PWM 듀티 30% 동작 시 모터 드라이버 과열 / 근본: firmware/laser_ctrl.c:142 PID 게인이 정격 부하 기준으로만 튜닝됨`
   - 개선 대책: `1) 단기: 듀티 25% 제한 + 냉각 시간 +0.5초 / 2) 장기: 부하 적응형 PID 알고리즘 적용`
   - 담당자: `김철수`
   - 조치 예정일: `2026-05-08` (펌웨어 수정+검증 +7영업일)
   - 조치 완료일: `` (빈칸 - 미완료)
   - 검증 결과: `검증 예정 - 8시간 부하시험 + 출력 안정도 ±2% 측정`
   - 비고: `일일업무보고 2026-04-15, 04-18 참조. 펌웨어 수정 리드타임 7일 가정`

6. **초안 제시**

```
| 행 | 장비       | 이슈 요약       | 원인 분석                  | 개선 대책                | 담당자 | 예정일     | 완료일 | 검증 결과              | 비고                       |
| 5  | 레이저 마킹기 | 출력 불안정    | 직접:PWM과열/근본:PID게인  | 1)단기:듀티25%/2)장기:PID적응 | 김철수 | 2026-05-08 |        | 검증 예정-8h부하시험   | 보고서 04-15,04-18 참조    |
| 7  | 레이저 마킹기 | 마킹 깊이 편차 | 직접:Z축 진동/근본:가이드 마모⚠ | 1)단기:속도-10%/2)장기:가이드 교체 | 미정 ⚠ | 2026-05-15 |        | 미검증                | 담당자 미정, 보고서 미발견 |
| 9  | 레이저 마킹기 | 통신 끊김      | 직접:RS485 노이즈/근본:접지 분리 미흡 | 1)단기:페라이트 코어/2)장기:접지 재설계 | 박영희 | 2026-04-30 | 2026-04-22 | 12시간 통신 무중단 확인 | 보고서 04-22 완료 기록 참조 |

⚠ 표시: 추정 또는 근거 부족 항목

이대로 시트에 반영할까요? 수정할 부분이 있나요?
```

7. **사용자 수정 요청**

```
7행 담당자를 '이수진'으로 지정해줘. 나머지는 OK
```

8. **최종 반영**
   - 7행 담당자만 '이수진'으로 수정
   - `mcp__google-sheets__batch_update_cells`로 3행 × 7컬럼 = 21셀 일괄 반영
   - 기존 K열 값은 그대로 보존, L~R열의 빈 셀에만 작성

9. **반영 결과 보고**

```
시트 반영 완료:
- 작성: 3개 행 × 7개 셀 (총 21셀)
- 시트 URL: https://docs.google.com/spreadsheets/d/1abcDEF.../edit#gid=0

후속 조치 필요 항목:
- 7행: 가이드 마모 추정 - 실제 마모도 측정 후 원인 확정 필요
- 7행: 검증 방법 미수립 - 담당자(이수진)와 협의 권장
- 9행: 조치 완료 기록 있으나 장기 대책(접지 재설계)은 미완료
```

## 결과

- 미작성 3개 행이 모두 채워짐
- 9행은 일일업무보고에서 완료 기록을 발견하여 조치 완료일이 자동 입력됨
- 7행은 담당자/근거가 부족하여 ⚠ 표시 + 비고 명시 → 사용자가 담당자만 수정 후 반영
- 모든 작성 항목에 근거(코드 파일, 보고서 날짜)가 비고에 명시되어 추적 가능
