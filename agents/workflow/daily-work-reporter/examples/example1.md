# Example 1: 기본 일일업무보고 작성 (명사형 종결 + 뎁스 구조 + 진행률)

## 시나리오

개발자가 퇴근 전에 하루 업무를 정리하여 Obsidian에 저장하는 기본적인 사용 예시입니다. 모든 항목은 명사형 종결과 뎁스(하위 항목) 구조로 작성됩니다.

## 대화 흐름

### 사용자 입력

```
오늘 로그인 API 버그 수정하고, 결제 모듈 코드 리뷰했어.
오후에는 팀 스탠드업 회의 참석했고, Docker 이미지 빌드 설정도 했어.
내일은 주문 페이지 리팩토링이랑 CI/CD 파이프라인 구축할 예정이야.
아 그리고 결제 모듈은 70% 정도 진행됐어.
```

### 에이전트 동작

1. **업무 내용 분류 + 진행률 파악**
   - 오늘 한 일: 로그인 API 버그 수정(100%), 결제 모듈 코드 리뷰(100%), Docker 이미지 빌드 설정(100%)
   - 진행 중인 일: 결제 모듈(70% - 사용자 명시)
   - 내일 할 일: 주문 페이지 리팩토링(0%), CI/CD 파이프라인 구축(0%)
   - 특이사항: 팀 스탠드업 회의 (진행률 없음)

2. **git 로그 확인** → 추가 커밋 없음

3. **기존 보고서 감지** → 오늘 날짜 보고서 없음 → CREATE 모드

4. **초안 생성 및 제시** (명사형 종결 + 뎁스 구조)

### 에이전트 출력 (초안)

```markdown
---
date: 2026-04-10
type: 일일업무보고
tags: [daily-report, work-log]
---

# 일일업무보고 - 2026년 04월 10일

## 오늘 한 일
- **로그인 오류 수정** `100/100%`
  - 로그인 시 화면 멈춤 현상 원인 파악 및 수정
  - 프로그램 간 소통 통로(API)에서 발생한 오류 해결
  - (로그인 API 버그 수정)
- **결제 코드 검토** `100/100%`
  - 동료 작성 결제 기능 코드 검토
  - 안전성/가독성 개선 의견 전달
  - (결제 모듈 코드 리뷰)
- **실행 환경 설정** `100/100%`
  - 프로그램이 어떤 컴퓨터에서든 동일하게 실행되도록 환경 패키징 설정
  - (Docker 이미지 빌드 설정)

## 진행 중인 일
- **결제 기능 개발** `70/100%`
  - 핵심 결제 로직 구현 완료
  - 남은 작업: 테스트 코드 작성 및 예외 처리
  - (결제 모듈 개발)

## 내일 할 일
- **주문 화면 정리** `0/100%`
  - 주문 페이지 내부 구조 정리 및 코드 개선 예정
  - 동작 유지한 채 내부 구조만 개선
  - (주문 페이지 리팩토링)
- **자동 배포 시스템 구축** `0/100%`
  - 코드 자동 검사 및 배포 시스템 구축 예정
  - (CI/CD 파이프라인 구축)

## 특이사항
- **팀 회의 참석**
  - 팀원 간 진행 상황 공유 및 논의
  - (팀 스탠드업 회의)
```

### 사용자 확인

```
사용자: "좋아, 저장해줘"
에이전트: "일일업무보고/2026-04-10.md 저장 완료."
```

# Example 2: 기존 보고서 업데이트 (진행률 갱신)

## 시나리오

오전에 이미 보고서를 작성했고, 오후에 추가 업무를 보고서에 업데이트하는 예시입니다. 기존 항목의 진행률이 변경된 경우 자동으로 갱신됩니다.

## 대화 흐름

### 사용자 입력

```
오후에 웹소켓 기능 구현하고, 보안 패치도 적용했어.
결제 모듈은 거의 다 됐어, 90% 정도.
```

### 에이전트 동작

1. 기존 보고서 감지 → UPDATE-PATCH 모드
2. 기존 "오늘 한 일" 3개 항목 유지
3. 새 항목 2개 추가 (웹소켓 100%, 보안 패치 100%)
4. **결제 모듈 진행률 70/100% → 90/100%로 갱신**

### 에이전트 출력 (병합된 초안)

기존 3개 항목 유지 + 새로 추가된 항목 + 진행률 갱신:

```
## 오늘 한 일
- **로그인 오류 수정** `100/100%`
  - [기존 내용 유지]
- **결제 코드 검토** `100/100%`
  - [기존 내용 유지]
- **실행 환경 설정** `100/100%`
  - [기존 내용 유지]
- **실시간 통신 기능 구현** `100/100%`
  - 서버-화면 간 실시간 양방향 통신 통로 구축
  - 채팅, 알림 등 실시간 기능에 활용
  - (웹소켓 기능 구현)
- **보안 조치 적용** `100/100%`
  - 외부 공격 방어를 위한 보안 조치 적용
  - (보안 패치 적용)

## 진행 중인 일
- **결제 기능 개발** `90/100%`
  - 핵심 결제 로직 구현 완료
  - 남은 작업: 테스트 코드 작성 및 예외 처리
  - (결제 모듈 개발)
  ↑ 진행률 갱신: 70/100% → 90/100%
```

### 에이전트 메시지

```
에이전트: "기존 보고서에 2개 항목 추가, 결제 모듈 진행률 70/100% → 90/100% 갱신. 확인 부탁드립니다."
```

# Example 3: 진행률 질문 흐름

## 시나리오

사용자가 진행률을 명시하지 않은 경우, 에이전트가 직접 질문하는 예시입니다.

## 대화 흐름

### 사용자 입력

```
오늘 인증 시스템 작업하고, 데이터베이스 마이그레이션 진행했어
```

### 에이전트 질문

```
에이전트: "두 가지 업무의 진행률을 확인할게요:
  1. 인증 시스템 작업은 대략 몇 % 정도 진행된 상태인가요?
  2. 데이터베이스 마이그레이션은 대략 몇 % 정도 진행된 상태인가요?"
```

### 사용자 답변

```
사용자: "인증은 절반 정도, 마이그레이션은 다 했어"
```

### 에이전트 진행률 적용

- 인증 시스템: "절반 정도" → 50%
- DB 마이그레이션: "다 했어" → 100%

### 에이전트 출력 (초안)

```
## 오늘 한 일
- **데이터 구조 업그레이드** `100/100%`
  - 데이터 저장 방식(구조) 업그레이드 완료
  - 새로운 기능 요구사항에 맞춘 저장소 개선
  - (데이터베이스 마이그레이션)

## 진행 중인 일
- **사용자 인증 시스템 개발** `50/100%`
  - 사용자 본인 확인 및 권한별 기능 접근 제어 시스템 구축 중
  - 남은 작업: 권한 관리 로직 및 테스트
  - (인증 시스템 개발)
```
