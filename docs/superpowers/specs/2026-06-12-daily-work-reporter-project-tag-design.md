# daily-work-reporter 프로젝트 폴더명 태그 설계

- 날짜: 2026-06-12
- 대상 에이전트: `agents/workflow/daily-work-reporter`
- 목적: 일일업무보고의 각 업무 항목에 그 작업이 속한 **프로젝트 폴더명**을 함께 기록하여, 하루에 여러 프로젝트를 오갈 때 항목별 소속 프로젝트를 구분 가능하게 한다.

## 배경

`daily-work-reporter`는 현재 작업 디렉토리에서 git 로그를 수집하고 Obsidian `일일업무보고/YYYY-MM-DD.md`에 보고서를 저장한다. 보고서 항목에는 카테고리 태그(`[FEAT]` 등), 진행률 바, 선택 메타데이터(공수·우선순위·식별자)가 있으나 **어느 프로젝트의 일인지**는 기록되지 않는다. 사용자는 항목 단위로 프로젝트 폴더명을 남기길 원한다.

## 결정 사항

### 1. 폴더명 캡처
- Phase 1(업무 수집)의 git 로그 수집 단계에서 함께 캡처한다.
- 1순위: `git rev-parse --show-toplevel` 성공 시 그 경로의 **basename**을 프로젝트 폴더명으로 사용 (예: `.../git_repo/custom_subagents` → `custom_subagents`).
- 2순위(폴백): git 저장소가 아니면 현재 작업 디렉토리의 basename.
- 둘 다 불가하면 **조용히 태그 생략**(기존 git 실패 처리와 동일하게 에러 미표시).

### 2. 태그 형식 & 위치
- 형식: `` `{폴더명}` `` — 백틱으로 감싼 **중괄호**. 카테고리 `[TAG]`·우선순위 `[P1]`의 대괄호와 시각적으로 구분된다.
- 위치: **진행률 바 바로 뒤**, 공수·우선순위 backtick 앞.
- 메타데이터 전체 순서: `**[카테고리] 제목**` → `` `N/100%` `` → `` `{프로젝트}` `` → `` `(공수)` `` → `` `[우선순위]` ``
- 예: `- **[FEAT] 로그인 기능 구현** ` + `` `100/100%` `{custom_subagents}` `(2h)` `[P1]` ``

### 3. 부착 범위
- 부착: **"오늘 한 일" / "진행 중인 일" / "내일 할 일"** 항목.
- 기본 생략: **"특이사항"** — 회의·이슈 등 프로젝트 무관 이벤트가 많음. 단, 사용자가 특정 프로젝트를 명시하면 부착 가능.

### 4. 소스 우선순위 (하루 여러 프로젝트 대응)
- 기본값: 자동 캡처한 현재 폴더명을 항목(git 로그 기반 포함)에 부착.
- 예외: 사용자가 특정 업무를 다른 프로젝트로 명시하면 **그 항목만** 명시값으로 덮어쓴다.

### 5. 병합 / UPDATE 모드 호환
- 기존 항목에 태그가 없으면 **그대로 보존**(과거 항목에 소급 적용하지 않음).
- 새로 추가하는 항목에만 태그 부착.
- 후위호환: 태그 없는 기존 보고서도 유효.

## 수정 대상 파일 (repo 4종 파일 컨벤션)

1. `daily-work-reporter.md` — Phase 1 캡처 로직, 보고서 형식, 메타데이터 표기 규칙, Output Standards, Constraints, Starting Instructions.
2. `examples/example1.md` — 예시 출력에 태그 반영.
3. `README.md` — Capabilities / Output Format / Version History(v1.2.0).
4. `agent.json` — capabilities·tags·version(1.2.0).

## 후속: weekly-work-reporter 연동 (2026-06-12 완료)

`weekly-work-reporter`(v1.3.0)가 daily 인라인 `{프로젝트}` 태그를 프로젝트 그룹핑 **최우선 소스**로 인식하도록 연동 완료:
- 항목 파싱(Phase 2 단계 5)에 인라인 `{폴더명}` 태그 추출 추가
- 그룹핑 우선순위 재정의: **인라인 태그 → H2 헤더 → frontmatter `project` → 괄호 키워드 → 제목 키워드 → "일반 업무"** (Phase 2 단계 6 + "프로젝트 그룹핑 규칙" 섹션 동기화)
- 동일 업무 다일 병합 시 가장 늦은 날짜의 태그 우선
- 진행률 파싱(`N/100%`) 무영향, 인라인 태그 없는 기존 보고서는 기존 우선순위로 동작(후위호환)
- 부속 파일(README v1.3.0, agent.json v1.3.0) 정합성 갱신

## 성공 기준

- 새 보고서의 "오늘 한 일/진행 중/내일 할 일" 항목에 `` `{폴더명}` `` 태그가 진행률 바 뒤에 표기된다.
- git 저장소가 아니거나 캡처 실패 시에도 보고서가 에러 없이 생성된다(태그만 생략).
- 기존 태그 없는 보고서를 업데이트해도 기존 항목이 손상되지 않는다.
- 사용자가 다른 프로젝트를 명시한 항목은 그 값으로 표기된다.
