# Weekly Work Reporter

## Overview

Obsidian에 기록된 일일업무보고(월~금, 평일만)를 자동으로 읽고 분석하여, 비전공자(중학생)도 이해할 수 있는 쉬운 한국어로 된 **상위 1% 품질의 주간업무보고**를 생성하는 에이전트입니다. 프로젝트별 업무 그룹핑, Executive Summary, 진행률 변화 추적, 다음 주 계획까지 체계적으로 정리하여 Obsidian에 저장합니다.

## Category

`workflow` - 주간 업무 보고 자동화 워크플로우

## Prerequisites

- Obsidian 앱 설치 및 실행
- [Obsidian Local REST API](https://github.com/coddingtonbear/obsidian-local-rest-api) 플러그인 설치 및 활성화
- Claude Code MCP 설정에 obsidian 서버 등록
- `일일업무보고/` 디렉토리에 `YYYY-MM-DD.md` 형식의 일일보고서가 존재
- Python 3 설치 (날짜 계산용)

## Capabilities

- Obsidian 일일업무보고 파일 자동 탐색 및 일괄 읽기
- 평일(월~금) 파일만 선별, 주말 파일 자동 제외
- 프로젝트별 업무 그룹핑 및 중복 업무 병합
- 주간 진행률 변화 추적 (시작값 -> 최종값)
- 경영진 보고 수준 Executive Summary 생성
- 리스크/블로커 종합 정리
- 다음 주 업무 계획 자동 도출
- 기술 용어를 비전문가도 이해할 수 있는 쉬운 한국어로 변환
- Obsidian 주간업무보고 디렉토리에 자동 저장

## Usage

### Basic Usage

```
이번 주 주간업무보고 작성해줘
```

에이전트가 자동으로:
1. 이번 주 월~금 날짜 계산
2. Obsidian에서 해당 날짜의 일일보고서 탐색
3. 프로젝트별 그룹핑 및 업무 집계
4. Executive Summary + 프로젝트별 상세 + 다음 주 계획 작성
5. 초안 제시 후 Obsidian에 저장

### Advanced Usage - 특정 주차 지정

```
4월 6일~10일 주간업무보고 만들어줘
```

지정한 날짜 범위의 일일보고서를 분석하여 주간보고서를 생성합니다.

### Advanced Usage - 부분 주

일일보고서가 일부 날짜만 존재하는 경우, 있는 자료만으로 주간보고서를 작성하고 누락 날짜를 통계에 명시합니다.

## Best Practices

1. **일일보고서를 꾸준히 작성하세요**: 매일 기록이 있을수록 주간보고서 품질이 높아집니다
2. **프로젝트 기반 H2를 사용하세요**: 일일보고서에서 `## 프로젝트명` 형태를 쓰면 주간보고서의 자동 그룹핑 정확도가 올라갑니다
3. **진행률을 명시하세요**: 일일보고서에 `N/100%` 형태로 진행률을 기재하면 주간 변화 추적이 정확해집니다
4. **금요일에 호출하세요**: 한 주 업무가 마무리된 금요일에 호출하면 가장 완전한 주간보고서가 생성됩니다
5. **초안을 꼭 확인하세요**: 저장 전에 보여주는 초안을 확인하고 수정 요청할 수 있습니다

## Limitations

| 가능 | 불가능 |
|---|---|
| Obsidian 일일보고서 기반 주간보고 생성 | 일일보고서 없이 자동 주간보고 생성 |
| 프로젝트별 업무 자동 그룹핑 | 다른 사람의 업무 보고서 분석 |
| 기술 용어 -> 쉬운 한국어 변환 | 비기술 업무의 자동 감지 |
| 진행률 변화 추적 | 진행률이 미기재된 항목의 정확한 추론 |
| 다음 주 계획 자동 도출 | 새로운 업무의 자동 예측/추가 |
| Obsidian에 주간보고서 저장 | Notion, Google Docs 등 타 서비스 지원 |

## Troubleshooting

### "Obsidian 볼트에 연결할 수 없습니다"
- Obsidian 앱이 실행 중인지 확인
- Local REST API 플러그인이 설치 및 활성화되어 있는지 확인
- 플러그인 설정에서 API 키가 올바른지 확인

### 일일보고서를 찾을 수 없을 때
- `일일업무보고/` 디렉토리에 `YYYY-MM-DD.md` 형식의 파일이 있는지 확인
- 파일명이 날짜 형식(예: `2026-04-10.md`)과 일치하는지 확인
- 다른 경로에 저장된 경우 에이전트가 자동으로 검색을 시도합니다

### Python 날짜 계산이 안 될 때
- Python 3가 설치되어 있는지 확인
- 설치되지 않은 경우 "4월 7일~11일" 형태로 직접 날짜 범위를 지정해주세요

### 프로젝트 그룹핑이 부정확할 때
- 일일보고서에서 프로젝트별 H2 헤더(예: `## ATL AXIS Frontend`)를 사용하면 정확도가 올라갑니다
- 초안 확인 시 그룹핑 수정을 요청할 수 있습니다

## Common Use Cases

1. **금요일 퇴근 전**: 한 주 업무를 자동으로 정리하여 깔끔한 주간보고서 생성
2. **월요일 주간회의 준비**: 지난주 주간보고서를 생성하여 회의 자료로 활용
3. **상사/팀장 보고용**: 비개발 직군 상사도 이해할 수 있는 형식의 보고서 작성
4. **개인 업무 회고**: 주간 단위로 업무 성과를 체계적으로 축적
