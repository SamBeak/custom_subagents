# Equipment Issue R&D Writer

## Overview

구글 시트 '장비 개발 이슈 문제점' 시트의 **연구개발(R&D) 작성 영역**을 자동으로 채워 넣는 에이전트입니다. 시트의 장비 구분(D열)과 이슈내용(K열)을 분석하고, 해당 장비 관련 프로젝트 코드/문서와 Obsidian의 '일일업무보고'를 종합하여 7개 항목(원인 분석, 개선 대책, 담당자, 조치 예정일, 조치 완료일, 검증 결과, 비고)을 작성합니다.

## Category

`workflow` - 장비 개발 이슈 R&D 작성 자동화

## Prerequisites

- Google Sheets MCP 서버 등록 (`mcp__google-sheets__*` 도구 사용 가능)
- 대상 스프레드시트에 대한 편집 권한
- Obsidian Local REST API 플러그인 + Obsidian MCP 서버 등록 (일일업무보고 참조용)
- 프로젝트 코드베이스가 작업 디렉토리에 존재 (장비별 모듈 탐색용)

## Capabilities

- '장비 개발 이슈 문제점' 시트 헤더 자동 매핑 + 미작성 행 식별
- 장비 구분(D열) + 이슈내용(K열) 컨텍스트 분석
- 프로젝트 코드/문서 검색을 통한 근거 자료 수집
- Obsidian 일일업무보고 검색·집계로 담당자/완료일 추정
- 7개 R&D 컬럼 자동 작성 (원인 분석, 개선 대책, 담당자, 조치 예정일, 조치 완료일, 검증 결과, 비고)
- 5Why 기반 원인 분석 + 단기/장기 개선 대책 분리 작성
- 추정 항목에 대한 근거/사유 비고 명시
- 기존 셀 값 보존 (덮어쓰기 방지)
- 사용자 확인 후 `batch_update_cells`로 일괄 반영

## Usage

### Basic Usage

```
장비 개발 이슈 문제점 시트의 R&D 칸 비어있는 행 채워줘.
시트 ID: 1abcDEF...xyz
```

에이전트가 자동으로:
1. 시트 헤더 매핑 + 미작성 행 식별
2. 각 행별 프로젝트 자료 + 일일업무보고 수집
3. 7개 R&D 컬럼 초안 작성
4. 사용자 확인 후 시트 반영

### Advanced Usage - 부분 처리

```
시트 5번~10번 행만 먼저 작성해줘
```

지정 범위만 처리하고 사용자 검수 후 나머지 진행 여부 확인.

### Advanced Usage - 특정 장비

```
'레이저 마킹기' 관련 이슈 행만 작성해줘
```

장비 구분(D열) 값으로 필터링하여 해당 장비 이슈만 처리.

## Output Format

### 작성 항목 형식

| 컬럼 | 형식 예시 |
|---|---|
| 원인 분석 | `직접: 모터 PWM 과열 / 근본: 방열 설계 마진 부족` |
| 개선 대책 | `1) 단기: 듀티 25% 제한 / 2) 장기: 방열판 면적 30% 확대` |
| 담당자 | `김철수` 또는 `미정 (배정 필요)` |
| 조치 예정일 | `2026-05-08` |
| 조치 완료일 | `2026-04-22` (미완료 시 빈칸) |
| 검증 결과 | `8시간 부하시험, 온도 ±1℃ 이내 확인` |
| 비고 | `일일업무보고 2026-04-15 참조, 부품 리드타임 7일 가정` |

### 초안 제시 형식

작성 후 표로 출력:

```
| 행 | 장비 구분 | 이슈 요약 | 원인 분석 | 개선 대책 | 담당자 | 조치 예정일 | ... |
| 5  | 레이저 마킹기 | 출력 불안정 | 직접:.../근본:... | 1)단기:.../2)장기:... | 김철수 | 2026-05-08 | ... |
```

근거 부족 셀은 ⚠ 표시.

## Best Practices

1. **시트 ID/URL을 명확히 전달하세요**: ID 또는 공유 URL을 정확히 알려주면 빠르게 시작합니다
2. **헤더가 정형화되어 있을수록 정확합니다**: '원인 분석', '개선 대책' 등 표준 명칭 사용 권장
3. **일일업무보고를 꾸준히 작성하세요**: 담당자/완료일 추정의 핵심 자료입니다
4. **장비 구분 값을 일관되게 입력하세요**: 동일 장비명을 표기 통일하면 검색 정확도가 올라갑니다
5. **시범 작성 → 검수 → 일괄 처리 순서를 권장합니다**: 대량 행은 먼저 5행 정도로 품질을 확인하세요

## Limitations

| 가능 | 불가능 |
|---|---|
| 빈 셀 자동 작성 | 기존 값 자동 덮어쓰기 (명시 요청 시만) |
| 코드/보고서 기반 근거 작성 | 외부 메일/메신저 자료 참조 |
| 5Why 기반 원인 분석 | 도면/회로도 시각 분석 |
| 일일업무보고에서 담당자 추정 | 인사 시스템 연동 |
| 추정 시 비고에 사유 명시 | 추정 정확도 100% 보장 |
| 한국어 명사형 종결 | 영어/일본어 작성 |

## Troubleshooting

### "스프레드시트에 접근할 수 없습니다"
- 시트 ID/URL이 정확한지 확인
- 인증된 Google 계정에 시트 편집 권한이 있는지 확인
- MCP 서버가 정상 동작 중인지 확인

### 헤더가 자동 매핑되지 않을 때
- 헤더 명칭이 표준 명칭(원인 분석/개선 대책 등)과 다른 경우 발생
- 에이전트가 후보 컬럼을 제시하면 직접 선택
- 또는 사용자가 정확한 컬럼 위치(예: L~R열)를 직접 지정

### 일일업무보고를 못 찾을 때
- Obsidian이 실행 중이고 Local REST API 플러그인이 활성화되어 있는지 확인
- 보고서가 '일일업무보고/' 디렉토리에 없으면 `simple_search`로 폴백 동작
- 그래도 못 찾으면 보고서 없이 진행 (담당자/완료일 정확도 하락)

### 작성된 내용이 추정에 가까울 때
- 비고 컬럼에 "근거 부족 - 코드 미확인" 등 사유가 명시됨
- 사용자가 추가 자료(설계 문서, 측정 데이터)를 제공하면 재작성 가능

### 시트 쓰기 후 일부만 반영되었을 때
- API 일시 오류로 인한 부분 실패 가능
- 에이전트가 행 단위 재시도를 자동 수행
- 최종 실패 행은 마크다운 표로 화면 출력 → 수동 복사 가능

## Common Use Cases

1. **주간/월간 R&D 회의 전 일괄 작성**: 미작성 이슈 행을 회의 전 일괄 채워 회의 효율 향상
2. **장비별 이슈 트래킹**: 특정 장비 관련 이슈만 필터링하여 집중 관리
3. **일일업무보고 ↔ 시트 연동**: 평소 작성하는 보고서를 R&D 기록으로 자동 연계
4. **품질 감사 대비**: 추적 가능한 근거(파일 경로, 보고서 날짜)와 함께 이슈 기록 유지
