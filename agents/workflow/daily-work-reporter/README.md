# Daily Work Reporter

## Overview

일일업무보고를 비전공자(중학생)도 이해할 수 있는 쉬운 한국어로 작성하여 Obsidian에 자동 저장하는 에이전트입니다. 각 업무 항목에 시각적 진행률 바를 표시하여 작업 상태를 한눈에 파악할 수 있어요. 기존 보고서가 있으면 기존 내용을 유지하면서 새로운 항목을 병합하고 진행률을 갱신해요.

## Category

`workflow` - 일일 업무 보고 자동화 워크플로우

## Prerequisites

- Obsidian 앱 설치 및 실행
- [Obsidian Local REST API](https://github.com/coddingtonbear/obsidian-local-rest-api) 플러그인 설치 및 활성화
- Claude Code MCP 설정에 obsidian 서버 등록
- (선택) Periodic Notes 또는 Daily Notes 플러그인 설정

## Capabilities

- 사용자 입력 기반 일일업무보고 작성
- 기술 용어를 비전문가도 이해할 수 있는 쉬운 한국어로 변환
- git 커밋 로그 자동 분석 및 보고서 반영
- Obsidian 데일리 노트 또는 전용 파일에 저장
- 기존 보고서 감지 시 중복 없이 병합 업데이트
- Obsidian 최근 변경사항 참조로 누락 업무 보완
- **카테고리 태그 자동 추론** (FEAT/BUG/REFACTOR/OPS/DOC/QA) — 주간보고 차트 정확도 향상
- **프로젝트 폴더명 태그 자동 부착** (`{폴더명}`) — 현재 작업 폴더를 자동 캡처하여 항목별 소속 프로젝트 구분
- **선택 메타데이터 지원**: 공수(시간), 우선순위, PR/JIRA 식별자, frontmatter 합계
- git 커밋 prefix → 카테고리 태그 자동 매핑

### 연속성 및 마감 연동 (v1.3.0)

- **전일 이월 제안**: 어제(최대 5일 역순) 보고서의 "내일 할 일"을 오늘 초안에 반영할지 물어봅니다. 승인한 항목만 포함됩니다.
- **진행률 연속성**: 어제 "진행 중"이던 업무는 어제 진행률을 기준으로 질문하며, 진행률이 후퇴하면 한 번 확인합니다.
- **마감 보호**: `/일일마감`으로 확정(`status: confirmed`)된 보고서는 명시적 승인 없이 수정하지 않으며, 수정 후 재확정을 안내합니다.

## Usage

### Basic Usage

```
오늘 로그인 API 버그 수정하고, 결제 모듈 코드 리뷰했어.
내일은 주문 페이지 리팩토링 할 예정이야.
```

에이전트가 자동으로:
1. 기술 용어를 쉬운 한국어로 변환
2. 4개 섹션으로 구조화
3. Obsidian에 저장

### Advanced Usage - 업데이트

이미 오늘 보고서가 있는 상태에서:

```
오후에 CI/CD 파이프라인 설정하고 팀 스탠드업 회의 참석했어
```

에이전트가 기존 보고서를 감지하고, 새 항목만 추가해요.

### Advanced Usage - 자동 분석

```
오늘 한 거 정리해줘
```

git 커밋 로그와 Obsidian 변경사항을 자동 분석하여 보고서를 생성해요.

## Output Format

```markdown
---
date: 2026-04-10
type: 일일업무보고
tags: [daily-report, work-log]
---

# 일일업무보고 - 2026년 04월 10일

## 오늘 한 일
- **로그인 오류 수정** `100/100%` `{shop_backend}`
  - 로그인 시 화면 멈춤 현상 원인 파악 및 수정
  - 서버 응답 대기 시간 초과 문제 해결
  - (session timeout 버그 수정)
- **결제 코드 검토** `100/100%` `{shop_backend}`
  - 동료 작성 결제 기능 코드 검토
  - 안전성/가독성 개선 의견 전달
  - (결제 모듈 코드 리뷰)

## 진행 중인 일
- **결제 기능 개발** `70/100%` `{shop_backend}`
  - 핵심 결제 로직 구현 완료
  - 남은 작업: 테스트 코드 작성 및 예외 처리
  - (결제 모듈 개발)

## 내일 할 일
- **주문 페이지 정리** `0/100%` `{shop_backend}`
  - 주문 화면 내부 구조 정리 및 코드 개선 예정
  - (주문 페이지 리팩토링)

## 특이사항
- 없음
```

## Best Practices

1. **구체적으로 입력하세요**: "버그 수정" 보다 "로그인 API에서 세션 타임아웃 버그 수정"이 더 좋은 보고서를 만들어요
2. **카테고리를 구분해주세요**: "오늘 한 일"과 "진행 중인 일"을 구분해서 말해주면 더 정확해요
3. **하루 중 여러 번 호출하세요**: 오전/오후 나눠서 호출하면 기존 보고서에 자동으로 병합돼요
4. **git 저장소에서 호출하세요**: git 로그를 자동으로 분석하여 누락된 업무를 보충해줘요
5. **초안을 꼭 확인하세요**: 저장 전에 보여주는 초안을 확인하고 수정 요청할 수 있어요

## Limitations

| 가능 | 불가능 |
|---|---|
| 사용자 입력 기반 보고서 작성 | 자동으로 모든 업무를 감지 |
| 기술 용어 → 쉬운 한국어 변환 | 비기술 업무의 자동 감지 |
| git 커밋 로그 분석 | 다른 사람의 커밋 분석 |
| 기존 보고서 병합 업데이트 | 과거 날짜 보고서 자동 생성 |
| Obsidian 데일리 노트 연동 | Notion, Google Docs 등 타 서비스 지원 |
| 해요체 한국어 보고서 | 영어/일본어 보고서 |

## Troubleshooting

### "Obsidian 볼트에 연결할 수 없어요"
- Obsidian 앱이 실행 중인지 확인
- Local REST API 플러그인이 설치 및 활성화되어 있는지 확인
- 플러그인 설정에서 API 키가 올바른지 확인

### 보고서가 데일리 노트에 저장되지 않을 때
- Obsidian Periodic Notes 또는 Daily Notes 플러그인이 설정되어 있는지 확인
- 미설정시 `일일업무보고/YYYY-MM-DD.md` 경로에 자동으로 생성돼요

### 기존 보고서가 덮어쓰기 될까봐 걱정될 때
- 이 에이전트는 기존 내용을 절대 삭제하지 않아요
- 새 항목만 추가하고, 중복은 자동 제거돼요
- 저장 전 반드시 초안을 보여주므로 확인 후 저장할 수 있어요

### git 로그가 반영되지 않을 때
- 현재 디렉토리가 git 저장소인지 확인
- 오늘 날짜의 커밋이 있는지 확인

## Common Use Cases

1. **퇴근 전 업무 정리**: 하루 끝에 업무 내용을 간단히 말하면 깔끔한 보고서로 변환
2. **점심 후 중간 업데이트**: 오전 업무를 먼저 보고하고, 오후에 추가 업무를 병합
3. **팀 공유용 보고서**: 비개발 직군 팀원도 이해할 수 있는 형식으로 작성
4. **개인 업무 로그**: Obsidian에 체계적으로 업무 기록을 축적

- **v1.3.0** - 전일 보고서 연속성·마감 보호
  - 전일 보고서 참조(Phase 1.5): "내일 할 일" 이월 후보 제안 — 사용자 승인 항목만 반영

- **v1.2.0** - 프로젝트 폴더명 태그 추가
  - 각 업무 항목에 프로젝트 폴더명 태그(`{폴더명}`) 자동 부착 — 항목별 소속 프로젝트 구분
  - 폴더명 캡처: `git rev-parse --show-toplevel` basename → 폴백 시 현재 작업 디렉토리 basename
  - 진행률 바 바로 뒤, 공수·우선순위 backtick 앞에 위치
  - "오늘 한 일/진행 중/내일 할 일" 항목에 부착, "특이사항"은 기본 제외
  - 사용자가 다른 프로젝트 명시 시 해당 항목만 그 값으로 덮어씀 (하루 여러 프로젝트 대응)
  - 캡처 실패 시 조용히 생략 — 후위호환(기존 보고서 형식 그대로 유효)

- **v1.1.0** - 메타데이터 보강 (주간보고 차트 정확도 향상)
  - 카테고리 태그 자동 추론 (`[FEAT]/[BUG]/[REFACTOR]/[OPS]/[DOC]/[QA]`)
  - git 커밋 prefix(feat/fix/refactor/docs/test/chore) → 카테고리 태그 매핑
  - 사용자 명시 시 공수(`(2h)`, `(0.5d)`)·우선순위(`[P0]`, `[Critical]`) backtick 표기
  - 사용자·git 로그에서 PR/JIRA/이슈 식별자 자동 추출 후 추적성 라인 추가
  - frontmatter `total_hours`/`focus_project` 선택 필드 지원
  - 모든 신규 메타데이터는 후위호환 — 기존 보고서 형식도 그대로 유효
