# PM Board Manager 사용 예시

## 예시 1: 갱신 실행 모드 (이전 보드 존재)

### 사용자 입력

```
PM 점검해줘
```

### 에이전트 처리 과정

1. **분석 기간 결정**
   - 미지정 → 최근 영업일~오늘 계산 (Python)
   - 이전 보드 `period_analyzed`가 2026-05-12~05-18 → 이번 분석 구간: 2026-05-19 ~ 2026-05-25

2. **이전 보드 읽기** → `PM보드/PM-Board.md` 존재 → **갱신 실행 모드**
   - 이전 보드의 열린 리스크: "윈도우 보안 권한 호환성"(최초 발견 05/16), "OPEN 파형 2ms 지연"(외부 협의)
   - 이전 보드의 "다음 액션": 자동 로그인 빌드 검증, 외부 협의 진행 등

3. **일일보고 수집** → 2026-05-19 ~ 2026-05-25 일일보고 읽기

4. **시간축 분석**
   - 약속 이행 추적: 이전 "다음 액션" 8건 중 6건 이행, "외부 협의 진행"은 5일째 미언급 → 미이행
   - 정체 감지: "자동 로그인 윈도우 호환성" 90%가 05/18 이후 변동 없음 → 5영업일 정체
   - 리스크 에이징: "윈도우 보안 권한" 경과 9일, 진전 없음 → 악화로 전이
   - WIP: 진행 중 9건 → 임계(8) 초과

5. **포트폴리오 집계** + 보드 초안 작성 → 사용자 확인 → 저장

### 출력 (PM보드/PM-Board.md)

```markdown
---
type: PM보드
updated: 2026-05-25
period_analyzed: 2026-05-19 ~ 2026-05-25
sources_daily: [2026-05-19, 2026-05-20, 2026-05-21, 2026-05-22, 2026-05-25]
config: { 정체임계일: 3, WIP임계: 8 }
tags: [pm-board, management]
---
```

## 예시 2: 초기 실행 모드 (이전 보드 없음)

### 사용자 입력

```
PM 보드 만들어줘
```

### 에이전트 처리 과정

1. `PM보드/PM-Board.md` 읽기 → 404 → **초기 실행 모드**
2. 최근 영업일 일일보고 수집
3. 모든 리스크·정체 후보를 **신규**로 베이스라인 설정 (경과일·정체 일수 = 0)
4. 약속 이행률은 추적할 이전 약속이 없으므로 "베이스라인" 표기
5. 보드 신규 생성 (`obsidian_append_content`)

### 출력 (요약)

```markdown
## 0. 한눈에 (PM 총평)
- 베이스라인 생성 · 정체 0건 · 열린 리스크 2건(신규) · WIP 7건
- 최초 PM 보드 생성 — 다음 점검부터 약속 이행률·경과일 누적 추적 시작

## 5. 변경 이력 (Changelog)
### 2026-05-25 갱신
- 최초 생성 — 리스크 2건 베이스라인 등록, 다음 실행부터 시간축 추적
```

## 예시 3: 부분 데이터 처리

### 사용자 입력

```
이번 주 PM 점검해줘
```

### 상황

이번 주 일일보고가 월·화·금만 존재 (수·목 누락)

### 에이전트 동작

1. 존재하는 3일분만으로 분석 진행
2. 보드 frontmatter `sources_daily`에 존재하는 날짜만 기록
3. 변경 이력에 "수·목 일일보고 누락 — 해당 구간 진행률 변동 추정 제외" 명시
4. 정체 일수 계산 시 누락일은 보수적으로 처리 (정체 과대 산정 방지)
