# Team Lead Weekly Briefing

팀장이 주간업무보고 회의에서 **구두로 말해야 할 내용**을 자동으로 정리해 주는 에이전트.

Obsidian에 저장된 주간업무보고(또는 일일업무보고)를 분석하여, 팀장 관점의 브리핑 스크립트를 `보고/` 폴더에 생성합니다.

## 특징

- **"문서형 보고서"가 아니라 "회의 발화용 스크립트"**
- 각 항목을 `포인트(요점) + 멘트(구어체 발화문)` 2단 구조로 제공
- 주간보고와 일일보고를 **교차 분석**하여 요약에 묻힌 디테일 복원
- 팀장 관점 6개 섹션: Highlights / Ongoing / Risks / Asks / Credit / 다음 주 포커스

## 입력

- 주간업무보고 파일 경로 (예: `주간업무보고/2026-W16.md`)
- 일일업무보고 파일 경로들 (예: `일일업무보고/2026-04-20.md` ~ `2026-04-24.md`)
- 또는 주차 지정 (예: "이번 주", "2026-W16") → 월~금 평일 보고 자동 탐색

## 출력

- `보고/YYYY-WNN-팀장브리핑.md` 파일 (Obsidian 볼트)

## 전제 조건

- Obsidian Local REST API 플러그인 실행 중
- `일일업무보고/`, `주간업무보고/` 디렉토리 존재
- (선택) `보고/` 디렉토리 — 없으면 자동 생성됨

## 사용 예시

```
"주간업무보고/2026-W16.md와 이번 주 일일보고로 팀장 브리핑 만들어줘"
"이번 주 팀장 브리핑 작성"
"일일업무보고/2026-04-22.md만 보고 브리핑 초안 만들어줘"
```

## 관련 에이전트

- `daily-work-reporter` — 일일업무보고 생성
- `weekly-work-reporter` — 주간업무보고 생성 (이 에이전트의 주요 입력 소스)
